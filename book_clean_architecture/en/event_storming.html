Event Storming

How to gather knowledge in one day that took months to accumulate.

Event Storming is a workshop format invented by Alberto Brandolini. Business experts and developers gather around a huge wall with sticky notes.

The main rule: we talk about what happened (Events), not about tables and classes.

Sticky Note Legend

Domain Event

Orange
Event (Past Tense)
"Invoice Paid"

Command

Blue
Action (Intent)
"Pay Invoice"

Policy

Purple
Rule (Whenever...)
"If overdue > 3 days"

Aggregate

Yellow
Object
"Order"

Why is this needed?

  • To find Bounded Contexts: where the language changes or the event flow breaks.
  • To find Aggregates: yellow sticky notes appear naturally when we group commands.
  • To see business "blind spots" that no one knew about.
